## What this is
macOS menu-bar **clipboard history manager**. Watches the system clipboard, stores text + image history to disk, and a **⌘⇧V** floating panel (fuzzy search · list · detail) finds and pastes past clips. Phase 2 adds an **actions** layer: per-item pipelines of typed steps (`script` / `ai`) that transform the text and paste the result, plus **⌃⇧V** to transform the current selection in place.

Successor to **AI Replace** (`../ai-replace`). Bundle ID `io.github.macrosak.recallyx`. Built in commit-sized slices.

> **`docs.local/`** (gitignored) holds implementation plans, design docs, and design references for your own use. **Never reference it anywhere that gets committed or published** — not in commit messages, PR descriptions, README, or any tracked `.md` file. This note is the only place it's named.

- **⌘⇧V** → history panel: search clipboard, ↵ pastes the selected clip, ⇥ opens its action menu.
- **⌃⇧V** (Phase 2) → grab the current selection, push to history, open the panel on its action menu (the AI-Replace replacement).

## Build
- `./scripts/bundle.sh` → `Recallyx.app` (uses `Recallyx Dev` identity if present, else ad-hoc). Honors `RECALLYX_VERSION` (set by CI) — stamps `CFBundleShortVersionString`/`CFBundleVersion`, stripping any `-suffix` to keep the plist numeric.
- `./scripts/make-dmg.sh` → `Recallyx-<version>-arm64.dmg` (built-in `hdiutil`, drag-to-install layout). Needs `Recallyx.app` built first; honors `RECALLYX_VERSION` for the filename.
- `./scripts/install.sh` — killalls running, copies to `~/Applications`, launches.
- macOS 13+, Command Line Tools only (no Xcode). SPM, zero deps.
- `swift build` for the library. **Run tests via `./scripts/test.sh`, not bare `swift test`.** The suite uses swift-testing (`import Testing`); under CLT that framework + `lib_TestingInterop.dylib` live off the default search path, so plain `swift test` fails with `no such module 'Testing'` (then dlopen errors). The wrapper adds the `-F`/rpath flags. CI has full Xcode, so its plain `swift test` works.

## CI / Releases (`.github/workflows/`)
- `pr-checks.yml` — runs `swift test` on PRs to `main` (macos-26 — gives the macOS 26 SDK with FoundationModels for the on-device AppleClient path).
- `release.yml` — on push to `main`: derive version `0.<N>` (`N = git rev-list HEAD --first-parent --count`, like AI-Replace's eywa scheme but `0.N` not `vN`), `swift test` gate → `bundle.sh` (ad-hoc) → `make-dmg.sh` → `gh release create` with the DMG. **No version-bump commit; the count is the version.** A `gh release view` guard makes re-runs idempotent.
- **Manual `workflow_dispatch`** (any branch via the native ref picker) → **pre-release** tagged `0.<N>-<branch-slug>.<short-sha>`, marked `--prerelease` so it never becomes "Latest".
- Signing is **ad-hoc** (no Apple secrets); notarization is a future drop-in gated on secrets (the seam lives in `bundle.sh`). Repo is public → `macos-26` runners are free.

## Source layout
The package splits into two targets: **`Sources/RecallyxCore/`** is a library of the portable, platform-free logic (Foundation/CryptoKit/FoundationModels/Security/os only — the iOS-reusable *brain*): AIProvider, Action, ActionMenuItem (the pure `BuiltinAction`/`ActionMenuItem` models, extracted from the SwiftUI ActionMenu), ActionRunner, the AI clients (Anthropic/Apple/Gemini/OpenAI+ModelCatalog/Ollama), AppState, FuzzyMatcher, HistoryItem, HistoryStore (+ ClipModel / PersistenceController — the programmatic Core Data stack), HistoryPanelViewModel (+ ClipTime), IconCatalog, KeychainStore, Log, ScriptRunner. Its public types are `public` only where the app/tests reference them. **`Sources/Recallyx/`** is the `Recallyx` executable (the macOS app) — all AppKit/SwiftUI/Carbon/AX/menu-bar/panel/paste code, plus `Settings.swift` and `Shortcut.swift` (Shortcut is Carbon/AppKit so Settings stays with it) and `PrivacyFilter`/`AppIconProvider` (await a later decoupling step). App files `import RecallyxCore`. Tests `@testable import` both. (Future: an iOS app reuses RecallyxCore.)
- `RecallyxApp.swift` — `@main` + `NSApplicationDelegate`. **All launch wiring lives in `applicationDidFinishLaunching`** (see Lessons — MenuBarExtra content is lazy).
- `AppState.swift` — `@MainActor ObservableObject` (status / lastError / historyCount).
- `StatusItemView.swift` — menu-bar dropdown. Includes **Search history** and **Transform selection** items whose key equivalents are derived live from the saved `Shortcut`s (observes `SettingsStore`; disabled hotkey → no key hint, item stays). The equivalents only fire while the menu is open, so they mirror — not double-trigger — the global Carbon hotkeys.
- `MenuBarIconImage.swift` — the menu-bar glyph: the brand mark (stacked clips, same viewBox-24 geometry as `BrandMark`) rendered as a resolution-independent **template** `NSImage` so macOS tints it for light/dark bars. `MenuBarIcon` (in `RecallyxApp.swift`) shows it at idle and swaps in an SF Symbol for working/success/error feedback.
- `Log.swift` — `os.Logger` (subsystem `io.github.macrosak.recallyx`) mirrored to stderr.
- `UsageJournal.swift` — `@MainActor final class`. **Opt-in, off-by-default, local-only** usage record: append-only JSON Lines to `~/Library/Application Support/Recallyx/usage.jsonl` (honors `RECALLYX_DATA_DIR`). `log(event, fields)` **no-ops immediately when disabled** (first check), else serializes one sorted-keys JSON line and appends **off the main thread** (`Task.detached`, tracked in `lastWrite` so tests await it). **Hard privacy rules:** no network code of any kind; never logs clip text / image bytes / clip file paths; search logs `queryLength: Int` only (never the text); errors log a category *string* (from the error type), never the raw message. **Size cap:** 2 MB; on an over-cap append it truncates-from-front (drops oldest lines) down to ~1.5 MB, keeping newest. Injectable file URL + `now` clock seam for hermetic tests. `clear()` deletes the file. Events: `panel_open{mode}`, `search{queryLength,resultCount}`, `paste{via,clipKind}`, `action_run{name,kind,stepTypes,provider,clipKind,custom}`, `action_error{name,category}`, `transform_selection{captured}`, `copy_selection{length}` (detail-pane ⌘C — length only, never the text). Unit-tested. Wired through the app delegate: panel_open in `HistoryPanelController.show`; search/paste in `HistoryPanelViewModel` (search on a ~400ms debounce); action_run/action_error in `AppDelegate.runAction`; transform_selection in `handleTransformSelection`.
- `HistoryItem.swift` — `HistoryItem` (stored record; new fields stay **Optional** for backward-compatible synthesized `Codable` decode since the store reseeds on any decode failure — `imageDimensions`, `pinned`/`isPinned`), `CapturedClip` (raw capture from the watcher), `ContentHash` (SHA-256 dedupe keys via CryptoKit).
- `HistoryStore.swift` — `@MainActor ObservableObject` owning the history. **Backed by Core Data** (`PersistenceController` wraps `NSPersistentCloudKitContainer`, CloudKit mirroring **OFF** in Phase 1d → plain local SQLite at `…/Recallyx/Recallyx.sqlite`; no entitlement/account needed). The model is built **programmatically** (`ClipModel.makeModel()` — no `.xcdatamodeld`, stays CLT/SPM-buildable) as one entity `ClipEntity` mirroring `HistoryItem`, modeled CloudKit-compatible (every attr optional/defaulted, no unique constraints — dedupe stays app-level via `ContentHash`). Public API unchanged (`add`/`bump`/`delete`/`clear`/`setPinned`/`cap`/`imageURL`/`flush`/`onChange`/`items`); the in-memory `items` array stays the canonical state (recency-ordered) and every mutation mirrors into Core Data via a debounced upsert-by-id `persist`. `items` stays pure recency order (pinned-first ordering is applied at the panel layer). Cap eviction (enforced on load too — `cap`'s `didSet` doesn't fire in `init`) **exempts pinned** (evicts oldest *unpinned* until within cap, leaving pins even above cap), orphan reconciliation (skipped on the reseed/corrupt-import path so backed-up PNGs survive). **Images stay on disk** as `images/<uuid>.png` (entity stores `imageFilename` only). **One-time JSON→Core Data migration:** on first launch when the store is empty and a legacy `history.json` exists → decode with the old logic, insert entities (ids/timestamps/pins preserved), then rename `history.json` → `history.json.bak`; idempotent (guarded by store-empty + the `.bak` rename), corrupt JSON tolerated (backed up to `.corrupt-*`, store stays empty). `init(…, inMemory:)` uses a `/dev/null` store for hermetic tests. See `ClipModel.swift` / `PersistenceController.swift`.
- `ClipboardWatcher.swift` — `Timer` polling `changeCount` (~0.3s). Privacy filter → classify image|text → capture frontmost app → `store.add`. `markSelfWrite()` self-write guard (keyed by pasteboard `changeCount`, recorded right after our own write — content hashes can't key this: re-encoded images never hash back to the captured bytes) so a paste-of-existing bumps rather than re-captures (AI/script results are *not* marked → re-enter as fresh top items). Image takes priority over text; TIFF/PNG normalized to PNG.
- `PrivacyFilter.swift` — pure `shouldCapture(types:captureSensitive:)` honoring `org.nspasteboard.{Concealed,Transient,AutoGenerated}Type` hints, + `isSkippableText`. Unit-tested.
- `AppIconProvider.swift` — `@MainActor` source-app icon resolver, memoized by bundle ID via `NSWorkspace.icon(forFile:)` (in-memory only).
- `Shortcut.swift` — `Shortcut` (Carbon keyCode + modifier mask + record-time `keyLabel` — captured via `characters(byApplyingModifiers: [])` so ⇧ never bakes into the label; lowercase stored, uppercased only for display). Derives `glyphs` (⌃⌥⇧⌘ order), `eventModifiers`/`keyEquivalent`/`keyboardShortcut` (nil when disabled), `from(event:)`; `Shortcut.validate` → `noModifier`/`conflict`/`systemReserved` (⌘Q/⌘W/⌘⇥ denylist). Unit-tested.
- `HotkeyManager.swift` — Carbon multi-hotkey, driven by the two `AppSettings` shortcuts (`showHistory` id=1, `transformSelection` id=2). `apply(_:_:)→ApplyResult` re-registers one hotkey live (per-id refs); `suspend()`/`resume(...)` unregister both while the Settings recorder captures keys (Carbon swallows registered combos before local monitors see them). The app delegate is the single mutation point: `applyShortcut` does Carbon-then-settings, so a failed registration never clobbers the live binding; launch failures land in `state.lastError`.
- `HistoryPanel.swift` / `HistoryPanelController.swift` — vibrancy `NSPanel` (760×562, `NSVisualEffectView .hudWindow`) + window controller (positions ~62% up the mouse screen, captures `sourceApp` to paste back into, routes ↑↓↵esc⇥ via a local keyDown monitor while typed chars reach the search field; ⌘1–9 quick-pastes the Nth visible clip in list mode via `vm.pasteItem(at:)`, and runs the Nth saved action in `.actions` mode via `vm.runSavedAction(at:)`). Cancels the previous task on every keystroke. Pin/Unpin acts locally + re-`ordered`s `allItems` and stays open (like Delete). `ClipTime` relative/clock formatting.

The text view is a `CopyInterceptingTextView` subclass: it overrides `copy(_:)` → `super.copy(_:)` then fires `onCopy(selectedSubstring)` (only for a non-empty selection) so a detail-pane ⌘C of *part* of a clip becomes a new stack clip. The `onCopy` closure is held on the Coordinator (refreshed in make/updateNSView) so it tracks the currently viewed clip across re-renders.
- **Copy-from-detail-panel** (detail-pane ⌘C → new clip, keep original selected): `LargeTextView.onCopy` → `DetailPaneView.onCopySelection` → `HistoryPanelView.onCopySelection` (wired only for the **list-mode** detail pane — the clip you're reading — so a stray ⌘C in action modes doesn't spawn clips) → `HistoryPanelController` injected `onCopySelection: (String, HistoryItem) -> HistoryItem?` → `AppDelegate.handleCopiedSelection`. The VM folds the clip into `allItems` (upsert-by-id so the store's content-hash dedupe-bump replaces in place, not duplicates), re-`ordered`s + re-filters for the current query, and re-pins `selectedIndex` to the **original** clip by id (falls back to top) — list-mode only (no-op otherwise). Logs a length-only `copy_selection{length}` usage event. VM method unit-tested; the AppKit copy hook is debug-harness-verified like other paste/AX paths.
- `ImagePreviewCache.swift` — `@MainActor` NSCache keyed by filename. `load(filename:url:)` decodes off-thread via `Task.detached` + `CGImageSourceCreateThumbnailAtIndex` capped at 340pt (the detail pane's content width) × backing scale. ### Phase 2 — actions / AI
- `Action.swift` — `Action { name, icon, steps: [Step] }`, `Step { type: .script|.ai, enabled, script, prompt, model? }` (generalizes AI Replace's `Preset`). `Action.defaults()` seeds the menu — AI text actions, a **dev script pack** (URL decode/encode, Base64 decode/encode, Decode JWT, Minify JSON — zero-config offline `python3` filters, placed after "Extract URLs" before the image actions), then image-friendly AI actions ("Extract text" / "Describe image"). `kindTag` → SCRIPT/AI. `Action.appendingMissingBuiltins(into:)` is the append-only, name-matched, idempotent merge behind the Actions-tab **"Restore built-in actions"** (↻) button — `AppSettings` only falls back to `defaults()` when the `actions` key is entirely absent, so this is how existing installs pick up newly shipped built-ins (and recover a default deleted by accident). Pure + unit-tested.
- `ActionRunner.swift` — `@MainActor`; threads text through enabled steps in order (`.script` → `ScriptRunner`, `.ai` → `AIClient` facade) via the shared `thread(_:current:)` loop. Two entry points share it: `run(_:on text:)` (text clips, unchanged) and `run(_:onImageData:)` (image clips — first enabled step must be `.ai`, gets the PNG → text, then remaining steps thread as text; `.script`-first throws `ActionError.scriptFirstOnImage`; empty pipeline → ""). The injected `runAI` seam is `(prompt, model?, input, imageData: Data?)`. Script/AI runners are **injectable** so tests are hermetic. A throwing step aborts before paste. **`ActionRunner.isEmptyResult(_:)`** (static, pure, unit-tested) reports whether a run produced nothing worth pasting (empty / whitespace-only); `AppDelegate.runAction` checks it for both the text and image paths and **skips the paste** when true (flashes "no output" + a "Action produced no output." notice, logs `action_error{category:emptyResult}`) — pasting "" would set the clipboard to "" and synth-⌘V over the user's current selection, silently wiping it. `ActionError.missingApiKey(AIProvider)` names the right provider; `.imageNotSupported` means "needs a vision-capable model" (cloud always, + per-model Ollama vision; `.apple` text-only). Unit-tested.
- `AIProvider.swift` — **multi-provider routing** + two shared AI-step helpers all clients route through: **`AIClientDefaults.maxOutputTokens` (4096)** — the single output-token cap the cloud clients request (OpenAI `max_completion_tokens`, Anthropic `max_tokens`, Gemini `generationConfig.maxOutputTokens`; raised from the old hardcoded 1000 that silently truncated long OCR/summaries — clients also log when the response's finish/stop reason signals length truncation), and **`applyPromptTemplate(_:text:)`** — fills the AI prompt: substitutes `{{TEXT}}` when present, else **appends** the input on its own line so a prompt that forgot the placeholder still sees the clip (instead of silently dropping it). All five clients (OpenAI/Anthropic/Gemini/Ollama/Apple) use it; the Custom… path's `buildCustomPrompt` ensures `{{TEXT}}` upstream so it never double-appends. `AIProvider.provider(for:)` picks by model-id prefix (`apple:*` → `.apple`, `ollama:*` → `.ollama`, `gemini*` → `.gemini`, `claude*` → `.anthropic`, else `.openai`) — no schema migration, OpenAI stays default. `AIClient.complete(prompt:model:input:imageData:)` dispatches to `OpenAIClient`/`AnthropicClient`/`GeminiClient`/`OllamaClient`/`AppleClient`; cloud providers (OpenAI/Anthropic/Gemini) read a keychain key (`keychain` is `nil` for the local `.ollama`/`.apple` and the facade skips the key check), Ollama reads the configured base-URL closure. With `imageData` set it runs a vision request gated by the **model-aware** `AIProvider.supportsVision(forModel:)` (cloud → true, `.apple` → false, `.ollama` → per-model via `isOllamaVisionModel` — substring allowlist `["llava","vision","bakllava","moondream","minicpm-v"]` on the prefix-stripped lowercased name, so `ollama:llava`/`ollama:llama3.2-vision` pass but `ollama:llama3.2`/`qwen2.5`/`mistral` throw `ActionError.imageNotSupported`). The instance `supportsVision` flag stays provider-level (false for `.ollama`/`.apple`) but the gate uses the static model-aware check. Cloud providers (`.openai`/`.anthropic`/`.gemini`) report `supportsVision == true`. Add a provider here (case + keychain/dispatch + vision rule). Unit-tested.
- `OpenAIClient.swift` / `AnthropicClient.swift` / `GeminiClient.swift` / `OllamaClient.swift` — raw `URLSession` clients (chat-completions / Messages API `https://api.anthropic.com/v1/messages` / Gemini `generateContent` `https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent?key=…` / local `\(baseURL)/api/generate`). **Gemini** is BYO-key (key as `?key=` query param), body `{"contents":[{"parts":[{"text": fullPrompt}]}]}`, vision appends an `inline_data` part (mime_type image/png, raw base64, no `data:` prefix), response text at `candidates[0].content.parts[0].text`; `GeminiError` mirrors `AnthropicError`. All three `complete` take an optional `imageData: Data?` (PNG bytes): non-nil → vision shape (OpenAI `image_url` data-URL; Anthropic `image` base64 source; **Ollama `"images": [<base64>]`** — raw standard base64, NO `data:` prefix); nil → the existing text shape. Ollama is **local, no auth, no key**, `stream:false`, 120s timeout, `ollama:` prefix stripped before send, friendly `notRunning`/`modelNotFound` (404) errors. **Ollama vision is per-model** (only multimodal locals like llava/llama3.2-vision; the facade gates non-vision models out). `ModelCatalog` (in `OpenAIClient.swift`) groups `.openAI`/`.anthropic`/`.gemini` (`gemini-3.5-flash`/`gemini-2.5-pro`/`gemini-3.1-flash-lite`)/`.ollama` (`ollama:llama3.2`/`qwen2.5`/`mistral` text + `ollama:llava`/`ollama:llama3.2-vision` vision)/`.apple` (`apple:on-device`); `.all = openAI + anthropic + gemini + ollama + apple`; `.default = gpt-4o-mini`. **Availability-aware pickers:** pure `ModelCatalog.groups(openAI:anthropic:gemini:ollama:apple:) -> [ModelGroup]` (flag-driven, fixed order OpenAI/Anthropic/Google Gemini/Ollama/Apple — unit-tested), `availableGroups()` computes the flags live (OpenAI/Anthropic/Gemini only when their keychain key is set, Apple only when `AppleClient.isAvailable`, Ollama **always** — local keyless v1 choice), and `groupsPreservingSelection(_:selected:)` appends a trailing single-item "Configured" group so a still-selected model from a now-unavailable provider doesn't blank the Picker. Both Settings model pickers route through these.
- `AppleClient.swift` — **on-device Apple Intelligence** via the FoundationModels system framework (macOS 26+), addressed `apple:*` (suffix ignored — the OS picks the model). **No key, no network, fully local; text-only in v1.** `complete` is `#if canImport(FoundationModels)`-guarded + runtime `#available(macOS 26, *)`-gated (app targets macOS 13+), with a throwing `#else`/unsupported-OS stub so it compiles on any SDK. Checks `SystemLanguageModel.default.availability` → friendly `AppleError.unavailable`/`unsupportedOS` when AI is off/undownloaded; runs via `LanguageModelSession().respond(to:).content`. `static var isAvailable` (macOS 26 + `availability == .available`; false where it can't import / OS < 26) gates the Apple group in the Settings model pickers. Unit-tested only via routing/guard (never calls the real model).
- `ScriptRunner.swift` / `KeychainStore.swift` / `Notifier.swift` — copied from AI Replace (env key `RECALLYX_SCRIPT`, keychain service `io.github.macrosak.recallyx`; accounts `openai-api-key` + `anthropic-api-key` + `gemini-api-key`).
- Action menu now shows built-ins → `Saved actions` divider → user actions (text clips only, with SCRIPT/AI tags). Running a saved action threads the clip text through `ActionRunner` and pastes the result (which re-enters history as a fresh top item — *not* marked self-copy). Settings General gains OpenAI, **Anthropic**, **and Google Gemini** sections (API key + Show/Test/Save each — Gemini Test hits `ModelCatalog.gemini.first!`), an **Ollama (local)** section (server-URL field, no key/Test), + a standalone **Default model** section (its own card after Ollama, before Shortcuts — a cross-provider setting, no longer nested under OpenAI) whose picker lists only `ModelCatalog.availableGroups()`. `AppSettings` extended with `defaultModel` + `actions` + `ollamaBaseURL` (default `http://localhost:11434`, `RECALLYX_OLLAMA_URL` env override; threaded to the facade via an `ActionRunner` closure like `defaultModel`).

- `SettingsActionsView.swift` (+ `IconCatalog`/`IconPickerView`) — the Actions tab: action list (add/delete/select; the selected row gets up/down chevrons to reorder, which is the order the panel's action menu shows) on the left, a step-pipeline editor on the right (name, icon picker, per-step type segmented Script/AI, enable toggle, body editor, model override, reorder/remove, Add step). The per-step **model override** picker lists `ModelCatalog.availableGroups()` (all available providers incl. Apple/Ollama, only configured cloud ones) after the leading "Use default" — same availability + selection-preserving logic as the Default-model picker. Edits write straight into `settingsStore.settings.actions`.
- **Ad-hoc AI in the panel.** The vm gains `.custom` and `.edit` modes. The action menu's **Custom…** entry (text clips) opens a one-off prompt column → ↵ runs a transient single-`ai`-step action. **Edit-before-run**: ⇥ on a highlighted saved action enters `.edit`, showing step 1's body editable; ⇥ paginates steps; ⌘↵ runs the modified *transient copy* (the saved action is untouched). Both go through the same `onRunAction` → `ActionRunner`. `CustomPromptColumn`/`EditStepsColumn` match the design. Focus moves to the editor in custom/edit modes, to the search field in list/actions.
- **The search field retargets by mode.** List mode filters clips ("Search clipboard…" / "N clips"); entering any action state clears it and switches to "Search actions…" / "N actions", filtering the menu (`filteredMenuItems`, order-preserving so the Saved-actions divider still groups). The clip query is stashed on ⇥ and restored on esc; `query`'s `didSet` routes to the active domain via `onQueryChanged`.

- `AccessibilityClient.swift` — trimmed copy of AI Replace's (read-only: selection capture + one-prompt-per-session permission flow; no write-back, since results paste via synth ⌘V). `captureSelection` reads `kAXSelectedText`; `captureSelectionViaCopy(markSelfWrite:)` is the Chromium/Gmail fallback — synth ⌘C, poll the pasteboard `changeCount` (~500ms), no bump ⇒ no selection. **It snapshots the user's clipboard first (`PasteboardSnapshot`, non-lossy: every type on every item) and RESTORES it after reading the copied selection**, then calls the injected `markSelfWrite` (wired to `watcher.markSelfWrite`) so the watcher's next tick treats the restore as our own write and won't recapture/bump it — without this, ⌃⇧V on a Chrome/Gmail selection permanently clobbered whatever the user had copied. The selection still reaches history via `handleTransformSelection`'s `store.add`, so restoring the live pasteboard is correct. `PasteboardSnapshot.capture/restore` is a small value type, unit-tested via a private named `NSPasteboard` (the synth-⌘C timing around it stays AppKit-bound). `handleTransformSelection` tries AX then the fallback, `store.add`s the clip to the top and **keeps the returned id**, then `historyPanel.showOnTopActions(focusing: id)` opens the panel already on that clip's action menu. **Targets the clip by id** (`HistoryPanelViewModel.openActionsOnTop(focusId:)`, falls back to `filtered[0]`) — the panel sorts pinned-first, so opening on `selectedIndex = 0` would land on the first pinned clip, not the captured selection.

**Phase 2 complete** — the full clipboard manager + actions/AI, ⌘⇧V and ⌃⇧V. Recallyx now supersedes AI Replace.

## Storage
`~/Library/Application Support/Recallyx/` — **`Recallyx.sqlite`** (the Core Data index; `NSPersistentCloudKitContainer`, CloudKit mirroring OFF in Phase 1d) + `images/<uuid>.png` (image payloads, still on disk — only the filename is in the entity). Pre-Core-Data installs migrate `history.json` → entities on first launch, then the JSON is renamed to `history.json.bak` (kept as a safety net). Only small settings/actions go in UserDefaults; history is in Core Data + on-disk images because images make it megabytes-large. Ordering is `max(createdAt, lastUsedAt)` descending (a bump refreshes `lastUsedAt`).

## Lessons carried over from AI Replace — don't relitigate
- **MenuBarExtra content is lazy.** Use `NSApplicationDelegate.applicationDidFinishLaunching` for launch wiring, never the content's `.task`.
- **Ad-hoc signing invalidates TCC every rebuild.** Stable identity via `scripts/create-signing-identity.sh` → grant survives rebuilds.
- **TCC stale entries survive System Settings toggles.** Fix: `tccutil reset Accessibility io.github.macrosak.recallyx`.
- **`open` doesn't relaunch a running app** — it foregrounds. `install.sh` does `killall` first.
- **Carbon `RegisterEventHotKey`** returns `eventHotKeyExistsErr=-9878` if the combo is taken globally.
- **Chromium/Electron silently drop `kAXSelectedText` writes** — re-read to verify, fall back to synthesized ⌘V at `.cghidEventTap`. (Phase 2 paste path.)
- **Chromium/Gmail don't expose `kAXSelectedText` reads either** (error or empty even with a selection) — fall back to synthesized ⌘C + pasteboard `changeCount` polling. AI Replace dodged this with a separate manual-copy hotkey (⌘⌥V); Recallyx synthesizes the copy itself.
- **OpenSSL 3 PBES2 p12 is rejected by macOS Security** — `create-signing-identity.sh` uses `/usr/bin/openssl` (LibreSSL).

## When the user reports a problem
1. App-side log: the `log stream` predicate above (info-level os_log is **not** persisted to disk — `log show` won't have it; use live `log stream`, or run the binary directly for the stderr mirror).
2. TCC log (⌃⇧V permission): `log show --predicate 'subsystem == "com.apple.TCC" AND eventMessage CONTAINS "recallyx"' --last 5m --info --style compact`. `Failed to match existing code requirement` ⇒ stale TCC entry — `tccutil reset Accessibility io.github.macrosak.recallyx`.
3. Codesign state: `codesign -dvvv Recallyx.app 2>&1 | grep -E "Authority|Signature"`. `Authority=Recallyx Dev` good; `Signature=adhoc` ⇒ TCC re-grant pain.
4. Re-bundle before testing hotkeys/UI: `swift build` updates `.build/`, but the `.app` binary is only refreshed by `scripts/bundle.sh`. Running a stale `.app` is a classic "my change didn't take" trap.

## Don't
- Add Xcode-only deps (no `#Preview` macros — Command Line Tools only).
- Move launch wiring onto MenuBarExtra content's `.task`.
- Run `tccutil reset` or `security add-trusted-cert` without explicit user confirmation.
- Test a code change against a stale `.app` — re-run `bundle.sh` first.
